# Test Code Review

用于评审单元测试、集成测试、回归测试、e2e / playwright 测试。

## 重点检查

- 是否覆盖对应 requirement / bug / acceptance item
- 断言是否真正验证业务结果，而不是只验证”没报错”
- 测试是否稳定、可重复
- 是否依赖过多隐性环境
- 是否存在大量无效 mock 或虚假覆盖
- 是否真的具备防回归能力
- 测试是否已实际运行通过（不只是”代码存在”）
- 如果依赖外部环境（DB、MQ 等）且未运行，是否在 progress 中明确标注原因

## 常见否决点

- 只有 happy path，没有关键失败分支
- 断言过弱，不能防回归
- 测试看似存在，但没有真实覆盖目标
- 测试高度依赖本地状态，无法稳定复现
- 测试已编写但从未运行，且未标注原因
- 测试失败被 `@Disabled` / `skip` 跳过，无修复计划

## 输出倾向

- 重点说明”这组测试到底防住了什么”
- 如果覆盖关系不清，明确指出证据缺口
- 区分”测试存在”和”测试运行通过”，两者不等价
