# Business Code Review

用于评审业务实现代码。

适用对象包括：

- Controller
- Service
- Domain / Entity
- Facade / Adapter
- Repository / DAO
- 前端业务实现代码

## 重点检查

- 业务逻辑是否正确
- 模块归属是否符合项目分层
- 实现是否遵守本模块 CLAUDE.md 的分层约束（common / impl / service 等）
- 是否引入不必要复杂度
- 是否存在明显重复、脆弱分支、命名歧义
- 是否会带来回归风险
- 是否有对应测试，且测试已实际运行通过
- 如果涉及跨模块编排，依赖方向是否会产生循环

## 常见否决点

- 实现不符合既有架构边界
- 逻辑错误或回归风险明显
- 用兼容/降级逻辑掩盖主链路问题
- 把代码放进部署壳或错误模块
- Controller / Service / 编排逻辑出现在 `*-cloud-service` 部署壳中
- 有测试但未运行，或测试失败被跳过

## 输出倾向

- 以 bug、风险、回归点为主
- 不要把代码评审写成方案重构长文，优先列出高风险问题
- 如果发现分层违规，明确指出应归属哪个模块层级
